    Kubara

    Kubara is a single binary CLI tool written in Go providing a lightweight framework for bootstrapping Kubernetes platforms with production-proven best practices very easily.

    This includes infrastructure provisioning, multi-tenancy setup, GitOps onboarding, and essential third-party tooling - all in a single reusable workflow.

    Start with the Bootstrap Your Platform Guide

    Short version of the bootstrap flow:

    1. kubara init --prep and prepare .env
    2. kubara init and prepare config.yaml
    3. kubara generate
    4. kubara bootstrap <cluster-name>

    What Is Kubara?

    Single binary CLI tool and lightweight framework for platform bootstrapping

    Kubara gives you a unified, reproducible way to deploy Kubernetes platforms with minimal effort and maximum consistency whether you're running on Cloud, Edge or Hybrid Setup.

    Kubara provides a reusable, extensible structure to build and operate your platform, including Terraform modules, Helm charts and secrets management, while keeping configuration simple and declarative.

    This includes infrastructure provisioning, multi-tenancy setup, GitOps onboarding, and essential third-party tooling in a single reusable GitOps workflow.

    Why Use Kubara?

    Setting up a secure, multi-cluster Kubernetes platform is hard.

    Kubara simplifies this with a single Go binary that standardizes the platform bootstrap process.

    • Generates required configuration and secrets
    • Templatizes and deploys pre-vetted infrastructure components
    • Bootstraps your platform using Argo CD and GitOps
    • Allows easy onboarding of new clusters and workloads

    All based on real-world usage at Schwarz Industry and the experience of multiple engineering teams - so you don't have to reinvent the wheel.

    How Does It Work?

    Kubara takes care of:

    1. Initial configuration via .env and config.yaml

      Prepare configuration and environment variables for the target platform setup.

    2. Rendering and deploying Terraform and Helm modules

      Generate and apply pre-vetted Terraform and Helm building blocks.

    3. Validating schemas and environment settings

      Validate generated output and environment-specific configuration before rollout.

    4. Bootstrapping the ControlPlane and Argo CD

      Bootstrap the ControlPlane and Argo CD using a GitOps-first workflow.

    5. Managing secrets (incl. External Secrets Operator)

      Use integrated secrets workflows and reusable patterns for secrets management.

    6. Adding additional worker clusters and workloads

      Onboard additional worker clusters and workloads using the same conventions.
